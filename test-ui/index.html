Integration Guide
1

Add an HTTP Request node

In your n8n workflow, add an HTTP Request node.

2

Configure the request

Method: POST   URL: https://your-worker.workers.dev/v1/images/generations

3

Add Authorization header

Header Name: Authorization   Value: Bearer YOUR_API_KEY

4

Set the body (JSON)

{ "prompt": "{{ $json.prompt }}", "model": "@cf/black-forest-labs/flux-1-schnell", "n": 1 }
5

Read the image

Access the base64 image from: {{ $json.data[0].b64_json }}

Use a Move Binary Data node to convert it to a file, or pass it directly to other nodes.

1

Add an HTTP module

In your Make.com scenario, add the HTTP → Make a request module.

2

Set URL and method

URL: https://your-worker.workers.dev/v1/images/generations

Method: POST

3

Add headers

Content-Type: application/json

Authorization: Bearer YOUR_API_KEY

4

Request body

{ "prompt": "{{1.prompt}}", "model": "@cf/black-forest-labs/flux-1-schnell" }
5

Parse response

The image is in data[0].b64_json as a base64 string.

Use Tools → Base64 Decode to convert it to binary, then send to Google Drive, email, etc.

// JavaScript / Node.js const response = await fetch('https://your-worker.workers.dev/v1/images/generations', { method: 'POST', headers: { 'Content-Type': 'application/json', 'Authorization': 'Bearer YOUR_API_KEY', }, body: JSON.stringify({ prompt: 'A futuristic cityscape at night', model: '@cf/black-forest-labs/flux-1-schnell', n: 1, num_inference_steps: 4, }), }); const data = await response.json(); const base64 = data.data[0].b64_json; // Convert to Blob for display/download const blob = await (await fetch(`data:image/png;base64,${base64}`)).blob();
# Python import requests, base64 response = requests.post( 'https://your-worker.workers.dev/v1/images/generations', headers={'Authorization': 'Bearer YOUR_API_KEY'}, json={'prompt': 'A futuristic cityscape', 'model': '@cf/black-forest-labs/flux-1-schnell'} ) b64 = response.json()['data'][0]['b64_json'] with open('image.png', 'wb') as f: f.write(base64.b64decode(b64))

curl https://your-worker.workers.dev/v1/models \ -H "Authorization: Bearer YOUR_API_KEY"
